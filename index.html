Single Page Application Basics

  • Single index.html page
  • Dynamically updates page based on user interaction - without reloading the entire page
  • Better User Experience
User Experience more similar to a native app - windows/mac/android/iphone/etc

Single Page Application Cons

  • Anyone browsing without JavaScript enabled will not get all the functionality
  • Search Engine Optimization (SEO)
  • It's not very common for people to browse without JS enabled anymore but it can happen
  • Search engines have a hard time crawling SPAs
    • Google has gotten better
    • Sometimes have to redirect bots to rendered content (prerender.io)

Single Page Application Considerations

Modularity

Small reusable pieces (aka components) across page

  • Header/Navigation Bar
  • Footer
  • Date Picker
  • Slideshow

Routing

  • Most SPA Frameworks/libraries include routing functionality
  • "Virtual" Pages
  • http://www.example.com/about
  • http://www.example.com/#/about
  • HTML5 History API - Use back & forward buttons

Angular2 Overview

How do we choose what framework or library to use?

  • Project Requirements
  • Personal Preference

Why did we choose Angular?

  • Handy CLI offered
  • Quick setup

Angular2 Basics

  • Angular CLI helps us scaffold our project quickly
  • Uses Typescript
    • Microsoft Developed
    • Statically typed superset of JavaScript
    • Compiled into JavaScript
    • .ts file extensions instead of .js
    • Can write JavaScript or Modern ES2015/ES6 in a TypeScript file
    • Makes using Angular2 easier

Angular CLI (Command Line Interface)

Documentation

  • CLI does things the "right" way according to Angular style guide
  • Install via npm
  • npm install -g angular-cli
Explain what a style guide is? (Naming is important in Angular)

What can we do with the CLI?

  • Create a new project 
    ng new PROJECT_NAME_HERE
  • Serve your project - so we can see it in the browser 
    ng serve

What can we do with the CLI?

Scaffold components, services, directives, etc.

  • Component 
    ng generate my-component-name-here
    OR 
    ng g component my-component-name-here
  • Service 
    ng g service my-service-name-here

What can we do with the CLI?

  • Build - Create dist directory with build artifacts 
    ng build
  • Deploy to GitHub Pages 
    ng github-pages:deploy --message "Optional commit message"
